## 第3页：项目背景 — 传统在线教育的三大痛点

**标题：** 为什么需要个性化学习？

**内容要点：**
- 🚫 学习资源千篇一律 — 所有学生看到相同的教材和题目，无法适配个体差异（有人基础好、有人零基础）
- 🧩 知识体系碎片化 — 知识点孤立排列，缺乏系统性的学习路径和前置依赖指导（不知道该先学什么后学什么）
- ⏰ 学习辅导缺乏交互 — 学生遇到问题只能查资料或等老师回复，AI时代应该有更好的解法

**结论：** 利用大语言模型构建多智能体系统，为每位学生提供个性化学习体验

| 数据层 | 用户/画像/路径/资源/聊天记录 | SQLite |

## 第6页：核心功能1 — 六维学生画像

**标题：** 基于对话的六维学生画像

**6个维度：**
1. 📚 知识基础 — 已掌握知识点的掌握程度(0.0-1.0)
2. 💡 认知风格 — 视觉型(图表) / 听觉型(讲解) / 动手型(代码)
3. ⚠️ 薄弱项 — 学生自述薄弱或反复出错的知识点
4. 🎯 学习目标 — 考试导向 / 项目实战 / 深入理解
5. ⏱️ 学习节奏 — 快速推进 / 边学边练 / 稳扎稳打
6. 🔥 兴趣方向 — NLP / CV / 强化学习等AI子领域

**构建方式：** 自然对话引导 + 每条消息LLM增量提取 → SQLite持久化

建议用雷达图或仪表盘风格展示6个维度

## 第8页：核心功能3 — 路径规划 + 智能辅导 + 效果评估

**标题：** 从规划到评估的完整闭环

**学习路径规划：**
- 知识图谱 + 前置依赖关系 → 拓扑排序 → 生成学习路径
- 以横向CSS卡片+SVG箭头展示，含知识点名称、预计时长、学习目标、配套资源
- 支持多门课程独立规划路径（Python/机器学习/深度学习）
- 节点状态可跟踪，进度自动持久化

**智能辅导"小智"（7步辅导法）：**
确认理解 → 一句话总结 → 直观类比 → 分层讲解 → ASCII图示 → 代码/公式 → 自测检验
学长风格，亲切易懂，支持emoji

**学习效果评估（5维度0-100分）：**
知识掌握 / 实践应用 / 学习投入 / 进步趋势 / 风险预警
每项含具体改进建议，按高/中/低优先级排序


## 第9页：技术创新亮点

**标题：** 7大技术创新

1. 🤖 **真正的多智能体架构** — 非单一大模型调用，11个专职Agent各有独立系统提示词和职责
2. 🗺️ **知识图谱驱动的路径规划** — 前置依赖拓扑排序 + 画像自适应调整 + 多课程独立管理
3. 🔍 **RAG防幻觉机制** — ChromaDB向量知识库，生成内容标注参考来源
4. 🛡️ **双层内容安全过滤** — 6类敏感词(~600词)，输入直接拦截，输出多类命中才拦截防误杀
5. 📊 **PPT代码高亮图片** — pygments渲染Monokai主题语法高亮PNG嵌入PPT，解决LLM无图像能力
6. ⚡ **流式SSE + 零抖动渲染** — Markdown实时渲染，Mermaid延迟加载防每15ms重解析
7. 🎨 **Mermaid图表交互** — 点击按钮渲染 + window.__mermaidCache缓存防滚动丢失 + 全屏遮罩放大
